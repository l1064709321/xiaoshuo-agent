##   起点白金大神作家风格参考 Skill

本技能覆盖近百位顶尖作家的风格定位与核心技法，提供从市场扫描、大纲设计、正文生成、质量审计到最终拍板的全流程创作支持。

---

## 一、动态调用流程

1. **分析需求**
   识别写作类型（玄幻/都市/悬疑/言情/历史/科幻等）、写作要素（开篇、人设、爽点、悬念、对话、战斗、感情线等）及创作阶段（大纲构思、正文推进、卡文续写、修改润色）。

2. **匹配流派**
   从作家风格库中筛选 1-3 个在该领域有突出建树的流派风格。优先匹配与用户需求最对症的流派，兼顾互补性。

3. **提取方法论**
   拆解该流派的核心技法，转化为可直接落地的创作建议。包括但不限于：节奏控制、人设公式、爽点排布、对话风格、悬念设置手法。

4. **给出范例**
   提供标志性段落或模拟文本，标注精妙之处，让用户直观感受该风格的笔法与气质。

## 三、半部小说续写引擎

### 3.1 工作流程

1. **作品解剖**
   对已有稿件进行五维拆解：
   - **主线与结构**：当前剧情处于什么阶段？主线的完成度如何？
   - **人物档案**：主角性格是否成型？配角有无弧光？关系网是否铺开？
   - **风格指纹**：文风偏爽文还是偏细腻？幽默还是严肃？战斗描写占比如何？
   - **伏笔与坑位**：已埋下哪些伏笔？哪些该收了？哪些还能挖？
   - **爽点密度**：最近三章的爽点分布是否合理？

2. **卡点诊断**
   判断卡文类型：剧情卡（不知道接下来发生什么）、人设卡（角色行为偏离设定）、节奏卡（不知道该快还是该慢）、设定卡（世界观有漏洞推不下去了）。

3. **流派外援匹配**
   根据作品类型和卡点，从流派库中匹配最对症的风格，引入该流派的解法。

4. **续写方案输出**
   - **作品体检报告**：五维评分 + 问题标注
   - **后续剧情路径**：提供 2-3 条可行的剧情发展方向，标注各自的优缺点
   - **可选续写示范**：选择最优路径，给出 500-1000 字的示范续写段落

### 3.2 卡点类型与对应解法

| 卡点类型 | 典型症状 | 推荐流派解法 |
|:---|:---|:---|
| **剧情枯竭** | 不知道接下来该写什么 | 引入“升级流”的突破-打脸循环；或“诡异流”的探索未知区域 |
| **节奏失衡** | 太拖或太赶，读者流失 | 参考“爽文教科书”的爽点排布密度；或“治愈系”的张弛节奏 |
| **人物崩塌** | 角色行为前后矛盾 | 用“凡人流”的谨慎算计逻辑重塑行为动机 |
| **设定漏洞** | 世界观出现矛盾 | 参考“规则流”的严密设定思维，给一切超自然力量定规矩 |
| **感情线僵硬** | 感情发展不自然 | 参考“克制的深情”流派，用细节替代直白表达 |

## 四、33 维审计系统

### 4.1 审计维度速查

| 类别 | 维度 | 核心检查项 |
|:---|:---|:---|
| **角色一致性** | 1-7 | 主角性格是否前后矛盾？配角是否沦为工具人？关系进展是否自然？智商是否在线？口头禅是否统一？外貌描写是否冲突？时间线是否错乱？ |
| **物资与战力** | 8-13 | 法宝/技能是否被遗忘？战力体系是否崩坏？物资数量是否对不上？资源使用是否合理？突破代价是否被削弱？货币体系是否混乱？ |
| **伏笔与逻辑** | 14-20 | 伏笔是否被遗忘？回收是否生硬？是否存在逻辑漏洞？巧合是否过多？时间线是否合理？信息获取是否合理？反派行为逻辑是否成立？ |
| **文风与表达** | 21-27 | AI味检测（机械句式/过度总结/对话像AI互谦）？描写是否冗长？战斗是否枯燥？情绪是否到位？幽默是否自然？ |
| **结构与节奏** | 28-32 | 章节结尾是否有钩子？爽点密度是否足够？开篇是否抓人？高潮是否燃？支线是否挤压主线？ |
| **大纲与偏离** | 33-36 | 当前剧情是否偏离大纲？对话是否过于密集？是否过多描写背景？是否过多加入旁白？ |

### 4.2 “去 AI 味”专项清洗

| AI 味典型症状 | 建议替换方向 |
|:---|:---|
| “他的心中不由得一惊” | 删除“不由得”，直接写动作：“他瞳孔微缩，后退半步。” |
| “这一切说来复杂，实则只发生在电光火石之间” | 直接删掉，用短句推进节奏。 |
| “他知道，自己必须变强” | 换成具体念头：“再这样下去，下次死的就不是别人了。” |
| “一股暖流涌上心头” | 用行动代替感受：“他别过脸，没让师父看见自己发红的眼眶。” |
| 每章结尾的“人生感悟” | 删掉，用悬念或动作收尾。 |

### 4.3 审稿报告格式

| 等级 | 标识 | 含义 | 处理方式 |
|:---|:---|:---|:---|
| 🔴 严重 | Critical | 影响阅读体验或逻辑自洽 | 立即修改 |
| 🟡 中等 | Warning | 削弱效果但不致命 | 本周期内处理 |
| 🟢 轻微 | Info | 可优化但非必须 | 择机优化 |
| ✅ 亮点 | Highlight | 写得好的地方 | 保持并强化 |

## 五、AI 编辑部流水线

### 5.1 工作室成员

| 角色 | 职能 | 输入 → 输出 |
|:---|:---|:---|
| **扫榜小哥** | 市场风向标 | 榜单/热搜 → 热门题材报告、脑洞趋势、竞品开篇分析 |
| **大纲老手** | 框架搭建 | 扫榜报告 + 你的创意 → 完整设定、分卷大纲、人物小传、伏笔清单 |
| **枪手代笔** | 正文量产 | 大纲 + 指定文风 → 章节正文（2000-4000字/章） |
| **毒舌编辑** | 质量审计 | 枪手草稿 → 33维审稿报告 |
| **改稿编辑** | 问题修正 | 审稿报告 + 原草稿 → 修订正文 + 修改说明 |

### 5.2 流水线运转流程
```
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│  立项    │  大纲    │  码字    │  审稿    │  返修    │  终审    │  拍板    │
├──────────┼──────────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│ 扫榜小哥 │ 大纲老手 │ 枪手代笔 │ 毒舌编辑 │ 改稿编辑 │ 毒舌编辑 │ 你       │
│ 市场摸底 │ 搭建框架 │ 批量出稿 │ 33维找茬 │ 问题修正 │ 再审一轮 │ 发布     │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
```
### 5.3 你的角色：老板

| 节点 | 你的动作 | 参考耗时 |
|:---|:---|:---|
| 立项 | 给一个点子，批准市场方向 | 10 分钟 |
| 大纲确认 | 检查大纲人设，有意见就提，没有就过 | 15-30 分钟 |
| 风格定调 | 扔一段你喜欢的文字作为文风参考 | 5 分钟 |
| 收货发布 | 审阅成品章节，行就发，不行就打回 | 每章 5 分钟 |

### 5.4 常用指令示例

| 场景 | 指令 |
|:---|:---|
| 启动全流程 | “工作室开工！题材：赛博修仙+家族流。主角背负‘修仙贷’的家族旁系。目标起点。” |
| 只要扫榜 | “扫榜小哥，最近一周都市异能有什么新书火了？” |
| 只要大纲 | “大纲老手，给我一份‘诡异修仙’前三卷大纲，主角能看到死亡倒计时。” |
| 卡文急救 | “我写了15万字卡住了，主角刚突破境界不知道干嘛，帮我看看。” |
| 只要审稿 | “毒舌编辑，这章总感觉有点水，帮我标一下哪里可以精简。” |
