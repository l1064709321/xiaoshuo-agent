## 一、原文语料库

> 核心升级：不再只存统计摘要，而是存**原文精选段落**。Prompt 直接塞原文做 few-shot，让模型从原文里学句式节奏、信息密度、断句习惯。

### 1.1 语料库规模

- **111位作者**，覆盖数据库中全部作家
- 每位作者 **6-8个精选段落**
- 按 **场景标签** 索引：battle / dialogue / environment / psychology / opening / climax / humor / suspense
- 每个段落有 **质量评分**（1-5星）和 **来源标注**

### 1.2 场景标签说明

| 标签 | 含义 | 适用场景 |
|:---|:---|:---|
| `battle` | 战斗/打斗 | 写战斗场面时参考 |
| `dialogue` | 对话/嘴炮 | 写对话时参考 |
| `environment` | 环境/场景描写 | 写环境时参考 |
| `psychology` | 心理/内心独白 | 写心理活动时参考 |
| `opening` | 开篇/出场 | 写开篇时参考 |
| `climax` | 高潮/燃点 | 写高潮时参考 |
| `humor` | 幽默/搞笑 | 写轻松段落时参考 |
| `suspense` | 悬疑/惊悚 | 写悬疑氛围时参考 |

### 1.3 语料库如何工作

当你调用 `deconstruct`、`ghostwrite`、`imitate` 时，系统会：

1. 根据你的需求自动匹配 **场景类型**
2. 从语料库中检索 **最相关的3段原文**
3. 将原文作为 **few-shot 参考** 注入 Prompt
4. 让模型从原文中学习 **句式节奏、信息密度、断句习惯**

**效果对比**：
- 旧版：`请用[作者]风格写` → 模板废话，AI味重
- 新版：直接塞3段[作者]原文 → 模型自己"看"出文风

### 1.4 查看语料库

```bash
# 列出所有作者
python3 main.py corpus

# 查看某作者的语料
python3 main.py corpus --author=[作者名]

# 按场景筛选
python3 main.py corpus --author=[作者名] --scene=battle

# 按关键词搜索
python3 main.py corpus --keyword=血
```

## 三、动态调用流程

1. **分析需求**：识别写作类型、写作要素及创作阶段。
2. **匹配作家**：从索引中筛选 1-3 位最对症的作家。
3. **检索原文**：从语料库中按场景标签检索该作家的精选段落。
4. **生成 Prompt**：将原文段落作为 few-shot 注入 Prompt，让模型从原文学文风。
5. **输出结果**：提供创作建议 + 原文参考 + 实操复用公式。

## 四、半部小说续写引擎

> 写了一二十万字突然卡住了？把半部稿子喂进来，引擎会自动拆解你的作品 DNA，结合大神技法给出能落地的续写方案。

### 4.1 工作流程

1. **吞下稿子**：把你写好的半部小说文本贴进来（建议至少 3 万字）。
2. **AI 解剖**：对稿子进行五个维度拆解——主线与结构、人物档案、风格指纹、伏笔与坑位、爽点密度。
3. **匹配大神外援**：根据作品类型和卡点，从作家索引中自动匹配 1-2 位最对症的大神。
4. **注入原文参考**：从语料库中检索该大神的精选段落，作为 few-shot 参考。
5. **输出续写方案**：给出作品体检报告、2-3 条后续情节路径、可选示范续写段落。

### 4.2 使用方式

```bash
# 卡文诊断
python3 main.py stuck --file=chapter.txt

# 文风仿写（带原文参考）
python3 main.py imitate --file=sample.txt --topic="修仙世界的拍卖会" --author=[作者名]
```

## 五、多维审计系统

> 写完一段自己读着挺顺？别急，扔进审核系统，它会从 36 个角度把毛病一个个揪出来。

### 5.1 审计的 36 个维度

| 类别 | 维度编号 | 检查内容 |
|:---|:---|:---|
| **角色一致性** | 1-7 | 主角性格前后矛盾、配角工具化、关系进展、智商在线、口头禅统一、外貌冲突、时间线错乱 |
| **物资与战力** | 8-13 | 法宝遗忘、战力崩坏、物资数量、资源使用、突破代价、货币体系 |
| **伏笔与逻辑** | 14-20 | 伏笔遗忘、回收生硬、逻辑漏洞、巧合过多、时间线、信息获取、反派逻辑 |
| **文风与表达** | 21-27 | AI味检测、描写冗长、战斗枯燥、情绪到位、幽默自然、对话质量、感官细节 |
| **结构与节奏** | 28-32 | 章节钩子、爽点密度、开篇抓人、高潮燃度、支线挤压 |
| **大纲与偏离** | 33-36 | 剧情偏离、对话密集、背景过多、旁白过多 |

### 5.2 "去 AI 味"专项清洗

| AI 味表达 | 建议替换方向 |
|:---|:---|
| "他的心中不由得一惊" | 删除"不由得"，直接写动作："他瞳孔微缩，后退半步。" |
| "仿佛失去了灵魂一般" | 用具体动作代替："他的手指在屏幕上一下一下地滑，划过去又划回来。" |
| "他知道，自己必须变强" | 换成具体念头："再这样下去，下次死的就不是别人了。" |
| "一股暖流涌上心头" | 用行动代替感受："他别过脸，没让师父看见自己发红的眼眶。" |
| "深吸一口气" | 改为"胸口起伏了一下"或"攥紧了拳头" |
| 每章结尾的"人生感悟" | 删掉，用悬念或动作收尾 |

### 5.3 审稿报告格式

- 🔴 **严重问题**（立即修改）
- 🟡 **中等问题**（本周期内处理）
- 🟢 **轻微问题**（择机优化）
- ✅ **亮点**（保持并强化）

## 六、AI 编辑部流水线

> 写书不用自己吭哧吭哧码字。这条流水线把一个网文工作室的整套班底全部 AI 化。你只需要当那个拍板的"老板"。

### 6.1 工作室成员

| 角色 | 外号 | 职责 | 输入 | 输出 |
|:---|:---|:---|:---|:---|
| **扫榜小哥** | 市场风向标 | 扒榜、分析趋势 | 起点/番茄榜单 | 热门题材报告、脑洞趋势 |
| **大纲老手** | 框架师 | 世界观与大纲设计 | 扫榜报告 + 你的创意 | 完整设定、分卷大纲、人物小传 |
| **枪手代笔** | 量产机 | 正文生成 | 大纲 + 原文语料参考 | 章节正文（2000-4000字/章） |
| **毒舌编辑** | 找茬专家 | 36维质量检查 | 枪手草稿 | 审稿报告 |
| **改稿编辑** | 救火队员 | 问题修正 | 审稿报告 + 原草稿 | 修订后正文 |

### 6.2 迭代写作流程

```
初稿生成（枪手代笔 + 原文语料参考）
    ↓
33维审计 + AI味检测（毒舌编辑）
    ↓
有问题？→ 改稿编辑修改 → 再审 → 再改...
    ↓
最多N轮，直到没有严重问题
    ↓
最终拍板（你确认发布）
```

### 6.3 老板怎么下指令

| 场景 | 指令示例 |
|:---|:---|
| 启动全流程 | `python3 main.py pipeline --concept="赛博修仙+家族流" --author=[作者名] --rounds=3` |
| 枪手代笔 | `python3 main.py ghostwrite --outline=大纲.txt --author=[作者名] --chapter=1 --words=3000` |
| 审稿 | `python3 main.py audit --file=chapter.txt` |
| 完整审计 | `python3 main.py full --file=chapter.txt` |
| 文风仿写 | `python3 main.py imitate --file=sample.txt --topic="拍卖会" --author=[作者名]` |
| 查看语料 | `python3 main.py corpus --author=[作者名] --scene=battle` |

### 7.6 工作室黑话速查

- **崩了**：战力/逻辑/人设前后矛盾
- **水了**：章节内容空洞，无剧情推进
- **毒点**：让读者极度不适的情节（送女、绿帽、圣母等）
- **AI味**：句式机械、对话客气、结尾总爱升华、"仿佛"泛滥
- **钩子**：章节结尾的悬念，勾引读者点下一章
- **爽点**：打脸、突破、震惊、反转，每1000-2000字至少一个
